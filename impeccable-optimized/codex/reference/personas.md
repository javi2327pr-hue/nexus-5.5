# Personas (Codex special ref)

```
LOADED BY: critique (always), shape (user discovery), onboard (activation)

PERSONA SPEC (for any persona):
  Name (memorable single-word)
  Role + context
  Goal (RIGHT NOW)
  Pressure (time? attention? mood?)
  Familiarity (category? product? conventions?)
  Constraints (device? bandwidth? a11y? language?)

8 DEFAULT PERSONAS:

1. "First Time"
   First visit; understand what + for-whom in <30s; zero familiarity
   Test: explained in 3 seconds?

2. "Returning Pro"
   Daily fluent user; in-out fast; knows shortcuts
   Test: common actions <2 clicks/keystrokes?

3. "Distracted Manager"
   30-sec meeting context; headline + status + decision
   Test: primary status identifiable in 3 seconds?

4. "Skeptical Reviewer"
   Evaluating vs alternatives; finds reasons to dismiss; 5 competitor tabs open
   Test: what sets this apart in first 10s? anti-refs applied?

5. "Mobile Commuter"
   Phone, subway, one hand; spotty conn; glare; one-thumb reach
   Test: touch 44x44? below-fold lazy? readable in bright light?

6. "Accessibility User"
   Screen reader OR keyboard-only OR high-zoom; friction-accumulated
   Test: semantic HTML? focus visible? ARIA right (not over)? contrast 4.5:1+?

7. "Power User Who Wants Customization"
   Long-term user; making it fit their workflow; expert
   Test: overrideable constraints? persistent prefs? advanced workflows reachable?

8. "Skeptical About AI"
   Tried 5 AI tools all "sparkles + purple gradient"; fatigue, low trust
   Test: distances from AI-marketing tells? anti-refs applied? slop test pass?

CUSTOM PERSONAS (for products):
  PRODUCT.md should have 2-3 specific:
    - Replace "First Time" with the specific first-time user
    - Replace "Returning Pro" with the specific daily user
    - Keep ONE outsider perspective (skeptic, a11y, mobile)

WALKTHROUGH EXERCISE:
  1. Pick 2-3 personas relevant to surface
  2. Imagine landing as this person (goal + pressure + familiarity)
  3. Walk forward: eyes first scan → primary action → hesitations
  4. Note stumbles (info gap, decision paralysis, friction)
  5. Compile per-persona findings; identify cross-persona patterns

PITFALLS:
  ❌ One persona only
  ❌ Walking the persona you ARE
  ❌ Skip accessibility persona
  ❌ Custom personas reading as marketing (need pressure/constraints/familiarity)
  ❌ Findings as opinions, not concrete fixes
```
