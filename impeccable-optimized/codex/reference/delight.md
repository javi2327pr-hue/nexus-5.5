# Delight (Codex)

```
Delight vs animate:
  animate = motion (functional + affordance + delight motion)
  delight = surprise + personality (motion is ONE way, also copy, illustration, empty states, sound, haptics)

Step 1: First-meaningful-action moment
  Identify the moment the user does the thing the product exists for
  Add: subtle celebration (color flash, soft sound, micro-haptic) + personality in copy
  NOT confetti (cheap)

Step 2: Empty states
  Replace "No items yet" with:
    - Hint what populated state could be
    - Voice matching PRODUCT.md tone
    - Clear next step CTA
    - Illustration if PRODUCT.md tone permits

Step 3: Loading + waiting moments
  - Communicate progress
  - Personality (tip / quote / custom illustration)
  - Under 3s OR explicit escape after that

Step 4: Error moments
  - Acknowledge what happened (not just "Error")
  - Suggest path forward
  - Don't blame user
  - DON'T be cute about serious failures (data loss, payment)

Step 5: Signature interaction (ONE per product)
  Linear's issue toggle, Notion's slash menu, Stripe's amount animation
  Pick ONE specific to what your product does
  Do NOT add five

Step 6: Register-aware
  Brand:   generous delight, signature moments, copy personality, illustration OK
  Product: surgical delight, celebrate user wins, otherwise stay invisible

Step 7: Route
  technical spectacle    → /impeccable overdrive
  motion specifically    → /impeccable animate
  copy carries delight   → /impeccable clarify (clarity IS delight)
  polish after           → /impeccable polish
```

## Anti-patterns
```
❌ Confetti everywhere (cheap)
❌ Delightful errors on serious failures
❌ Five signature interactions
❌ Loading tips that distract
❌ Easter eggs on high-cognitive-load surfaces
❌ Empty state illustration EVERY empty state
```
