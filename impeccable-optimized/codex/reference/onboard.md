# Onboard (Codex)

```
Step 1: Welcome / sign-up flow
  - Cut every field not required to get started
  - Social login first; email/password fallback
  - Single primary path (multiple = decision fatigue)
  - Progressive profile (ask non-critical info LATER)

Step 2: First-run state (no data)
  - Empty state designed as CTA, not placeholder
  - One clear action: create / connect / import / start
  - Show preview of populated state (illustration, demo data)
  - Skip "Take a tour" patterns — guide via actual UI

Step 3: Activation moment (the aha)
  Identify YOUR product's aha moment:
    - Notion: first page created
    - Linear: first issue moved to "In Progress"
    - Stripe: first test payment cleared
    - Figma: first frame on canvas
  Ensure nothing blocks it
  Celebrate (see /impeccable delight)

Step 4: Progressive disclosure (after activation)
  - Contextual tooltips (only when user lands on new surface)
  - Feature announcements (subtle, dismissible, NEVER modal)
  - Empty states with hints ("Try [feature] — does [outcome]")

Step 5: Register-aware
  Brand:   marketing → product handoff (signup IS the conversion)
  Product: single-path, fast, value-first

Step 6: Route
  empty/error/loading design   → /impeccable harden
  copy specifically            → /impeccable clarify
  activation celebration       → /impeccable delight
  final pass                   → /impeccable polish
```

## Anti-patterns
```
❌ Modal welcome with 5 slides
❌ "Take a tour" (95% skip rate)
❌ Demand profile completion before value
❌ Credit card before user sees value
❌ "No items" empty state (use as CTA)
❌ Tooltips on every element (overload)
```
