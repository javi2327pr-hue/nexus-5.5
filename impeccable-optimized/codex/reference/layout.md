# Layout (Codex)

```
Step 1: Spacing rhythm (vary across sections)
  Hero            → generous
  Content blocks  → moderate
  Lists/grids     → tight but breathable
  Scale: 4/8/12/16/24/32/48/64/96 — no orphan 17px values

Step 2: Hierarchy through scale + density
  Big things READ bigger
  Dense lists READ smaller
  Hierarchy = type size + information density per area

Step 3: Cards
  - NO nested cards (always wrong, collapse)
  - NO identical card grids (same size + icon + heading + text repeated)
  - Vary card sizes OR use editorial layout OR rethink

Step 4: Container restraint
  Most things don't need a container
  Remove containers that don't earn their place
  Watch for the wrap-everything reflex

Step 5: Grid + asymmetry balance
  Pure symmetry = safe
  Pure asymmetry = chaos
  ONE deliberate asymmetric moment per section
  (offset hero, larger first item, full-bleed escaping grid)

Step 6: Breakpoint behavior
  Desktop AND mobile
  REFLOW, don't shrink:
    Two-column → stacked, possibly reordered
    Wide hero → centered narrower
  Touch targets ≥44x44 at coarse pointer

Step 7: Register-aware
  Brand:   editorial composition, asymmetry allowed, vary aggressively
  Product: predictable grids, asymmetry only signals priority

Step 8: Route
  spacing fix → type hierarchy gap → /impeccable typeset
  breakpoint broken                 → /impeccable adapt
  final pass                        → /impeccable polish
  composition broken                → /impeccable critique
```

## Anti-patterns
```
❌ Same padding everywhere ("consistent" = monotonous)
❌ Nested cards
❌ Identical card grid
❌ Wrap-everything-in-container reflex
❌ Mobile = scaled-down desktop (reflow, don't shrink)
```
