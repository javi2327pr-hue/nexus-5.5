# Brand register (Antigravity)

## Applicability
Loaded by SKILL.md Setup when register = brand. Applies to: brand sites, landing pages, marketing surfaces, campaign pages, portfolios, long-form content, about pages.

## Stance
Communicate, not transact. Visitor's impression IS the thing being made. Span genres (tech, luxury, consumer, creative) — share stance, diverge in aesthetic lane.

## Constraints
1. Brand slop test passes (1st-order: theme/palette from category alone? 2nd-order: aesthetic family from category+anti-refs?)
2. Aesthetic lane named explicitly (Klim-specimen / Stripe-minimal / Liquid-Death-acid-maximalism / etc.)
3. Reflex-reject aesthetic lanes rejected: editorial-typographic on non-editorial brief, AI-dark-mode-with-purple, healthcare-cream-teal, crypto-neon-on-black
4. Typography: strict reflex-reject list (Fraunces, Cormorant, Inter, Plex, etc.) applied
5. Pairing matches brand type (editorial/tech/consumer/industrial/cultural)
6. Color: Committed/Full/Drenched freely available; Restrained is opt-in, not default
7. Layout: editorial composition, asymmetry allowed, vary rhythm aggressively
8. Motion: signature moments allowed, scroll-driven reveals OK
9. Image-led briefs: real or sourced imagery, not CSS scenery / filler

## Acceptance criteria for brand output
1. Aesthetic lane named before committing
2. 1st + 2nd-order slop tests pass
3. Restraint chosen with intent (not by default)
4. POV identifiable in 30s glance
5. Typography from outside reflex-reject (or justified)
6. Required imagery present (not replaced with decoration)
7. Existing brand identity preserved if applicable

## Failure modes
- Brief is genuinely categorical (e.g., "AI tool landing page") with no anti-references → ask user for anti-refs (highest leverage)
- Brand has identity already committed to reflex-rejected lane → preserve identity, document trade-off in DESIGN.md
- Greenfield with multiple plausible aesthetic lanes → present 2-3 options with visual probes (Codex) or descriptions (other harnesses)
- Image-led brief without imagery available → ask user for source OR explicitly note imagery gap in deliverable
