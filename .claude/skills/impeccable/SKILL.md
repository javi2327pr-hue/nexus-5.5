## Shared design laws (apply to every command, both registers)

### Color
- Use OKLCH. Reduce chroma as lightness approaches 0 or 100.
- Never `#000` or `#fff`. Tint every neutral toward the brand hue.
- Pick a strategy: **Restrained** / **Committed** / **Full palette** / **Drenched**. The "one accent ≤10%" rule is Restrained only.

### Theme
- Dark vs light is never a default. Write one sentence of physical scene; if it doesn't force the answer, add detail until it does.

### Typography
- Body line length 65-75ch.
- Weight contrast ≥1.25 between hierarchy steps.

### Layout
- Vary spacing for rhythm. Same padding everywhere = monotony.
- Cards are lazy; nested cards always wrong.
- Don't wrap everything in containers.

### Motion
- Don't animate CSS layout properties.
- Ease-out exponential (quart/quint/expo). No bounce, no elastic.

### Absolute bans (rewrite the element with different structure)
- Side-stripe borders (>1px colored)
- Gradient text (`background-clip: text`)
- Glassmorphism as default
- Hero-metric template
- Identical card grids
- Modal as first thought

### Copy
- Every word earns its place.
- No em dashes. Use commas, colons, semicolons, periods, parentheses. Also not `--`.

### The AI slop test (run twice)
- **1st-order**: theme+palette guessable from category alone? (observability→dark blue, healthcare→white+teal, finance→navy+gold) → rework.
- **2nd-order**: aesthetic family guessable from category+anti-references? → rework.

Full design laws + register-specific guidance: `reference/brand.md` / `reference/product.md`.

## Routing between commands (NEW in v3)

|---|---|

## Gate enforcement (NEW in v3 — fights compression)

Multi-gate commands (especially `craft`) require explicit user confirmation at each gate. The checkpoint script enforces this mechanically:

```bash
node {{scripts_path}}/checkpoint.mjs --gate=<n> --status=<confirmed|pending>
```

For `craft`, all 4 gates must be `confirmed` before any code is written. Skipping a gate returns non-zero.

## Routing rules

1. **No argument**: render commands table grouped by category, ask what to do.
2. **First word matches a command**: load `reference/<command>.md` and follow it. Remainder of argument is the target.
3. **First word doesn't match**: general design invocation. Apply Setup + shared design laws + register reference, using full argument as context.

If first word is `craft`, Setup still runs first, but `reference/craft.md` owns the rest of the flow. If Setup invokes `teach` as a blocker, finish `teach`, refresh context, then resume.

## Anti-patterns for this skill itself

- ❌ Skipping Setup ("I know what they want")
- ❌ Compressing craft gates 2-4 after shape confirmation (dominant failure mode)
- ❌ Editing the commands table by hand instead of `command-metadata.json`
- ❌ Switching command mid-task without telling the user
- ❌ Loading `live.md` (622L) when not in live mode
- ❌ Hardcoding `impeccable-asset-producer` invocation outside Codex (it's Codex-only)
